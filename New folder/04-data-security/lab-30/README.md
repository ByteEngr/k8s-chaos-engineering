## Lab 30 – Data: Data Security Recap and Recommendations

### Scenario
You are preparing a short **data protection review** after completing several data labs. Summarise **common failure modes** (exfil channels, insider risk, mis-sharing) and propose **three improvements** your organisation could implement next quarter.

### Investigation Objectives
- Synthesise lessons into **prioritised risks** (likelihood × impact).
- Propose **controls** across people, process, and technology.
- Identify **metrics** (DLP hit rates, time-to-contain, training completion).
- Draft a **non-technical summary** for leadership.

### Logs/Artifacts Description
You have access to:

- **Your own notes** from prior labs (synthetic recap prompts provided in class).

### Tasks
1. **Pick three themes**
   - Example: HTTPS exfil, external shares, database service accounts.
2. **Recommendations**
   - One paragraph each with owner and effort.
3. **Metrics**
   - How you would measure success.
4. **Roadmap**
   - 30/60/90 day plan.
5. **Risks of inaction**
   - One paragraph, factual tone.

### Questions
- Which improvement reduces risk fastest?
- What requires legal or privacy review first?
- How do you avoid “tool shopping” without process?

### Indicators of Compromise
Provide a **sample enterprise data IOC list** (generic):

- Unapproved cloud domains, suspicious share links, abnormal DB query sources.

### Expected Analyst Outcome
By the end of this lab, a junior analyst should be able to:

- Connect tactical investigations to a sustainable data security programme.
