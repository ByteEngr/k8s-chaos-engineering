## Lab 24 – Network: Perimeter Recap and Control Recommendations

### Scenario
You are summarising perimeter monitoring for a monthly review. You must **synthesise** findings from denied traffic, IDS alerts, and proxy blocks into **three actionable recommendations** for leadership (people, process, technology).

### Investigation Objectives
- Prioritise **top risks** seen across the week (synthetic summary data).
- Identify **control gaps** (logging, coverage, staffing).
- Propose **metrics** to track next month.
- Communicate without fear-mongering.

### Logs/Artifacts Description
You have access to:

- **Executive summary tables** (provided): top IDS signatures, top deny sources, top proxy categories.
- **One incident narrative** (short) requiring follow-up.

### Tasks
1. **Pick three themes**
   - Example: brute force attempts, phishing clicks, east-west gaps.
2. **Map to controls**
   - Prevent, detect, respond for each theme.
3. **Recommendations**
   - Specific changes with owners (network, IT, SOC).
4. **Metrics**
   - What to measure to show improvement.
5. **One-page brief**
   - Bullets suitable for a steering committee.

### Questions
- Which recommendation is quick win vs multi-quarter?
- How do you show value of monitoring when nothing “bad” happened?
- What would you not put in an executive summary?

### Indicators of Compromise
Include a **short IOC appendix** of the week:

- Top external IPs worth blocking or threat-intel review.

### Expected Analyst Outcome
By the end of this lab, a junior analyst should be able to:

- Turn operational telemetry into governance-ready recommendations.
